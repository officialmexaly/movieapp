DESIGN PRINCIPLES

1. GRADIENT-HEAVY              Use gradients liberally
2. DARK FOUNDATION             #0D0D0D base, light overlays for depth
3. SMOOTH MOTION               Elastic and expo easing curves
4. BOLD TYPOGRAPHY             Heavy weights (600, 700, 800, 900)
5. TRANSPARENT LAYERING        Opacity hierarchy instead of color changes
6. PRECISE SPACING             Consistent grid units
7. ACCESSIBILITY FIRST         WCAG AAA contrast, haptic feedback
8. MICRO-INTERACTIONS          Every interaction has feedback

DEVELOPER QUICK NOTES

Q: Exact colors important?
A: YES for brand colors. Purple/Blue/Cyan are carefully chosen.

Q: Match animation timings?
A: Aim for it, but 300-400ms is standard for most interactions.

Q: Use exact fonts?
A: SF Pro Display ideal, but system fonts with tight spacing work.

Q: Match spacing exactly?
A: YES for consistency. Use 4px base unit.

Q: Dark mode only?
A: For MVP yes. Light mode can be added later.

Q: Glow effects too subtle?
A: Increase blur radius in box-shadow, increase opacity.

Q: Animations feel sluggish?
A: Reduce duration by 100-200ms, use elasticOut curve.
